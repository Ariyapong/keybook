| | |
|---|---|
| **Status** | Draft, pending review |

| **Target** | v0.2 (Homebrew distribution + readline-style search input) |

---

## 1. Executive summary

v0.2 makes keybook **easier to install** and **easier to type into**, the two
biggest friction points uncovered after the v0.1 release. There are no schema
changes and no new data model — just one new distribution channel and a small
set of edit-input bindings the user expects from any terminal text field.

**Scope (firm):**
1. A **Homebrew tap** (`Ariyapong/homebrew-tap`) so `brew install ariyapong/tap/keybook` works on macOS without ever touching Node directly.
2. **Phase A readline editing** in the search input: word-back delete (`⌥⌫`, `⌃W`) and clear-query (`⌃U`). No cursor-position model — cursor stays pinned to the end of the query, same as v0.1.

**Out of scope for v0.2** (deferred — see §8): full cursor model (Phase B
readline), favorites, app filter, `keybook add`, more seed apps, Homebrew core
submission, standalone binary.

---

## 2. Goals & non-goals

### Goals
- A first-time macOS user can install keybook in **one command** without
  installing Node manually: `brew install ariyapong/tap/keybook`.
- Editing a multi-word search query feels **natural**: `⌥⌫` removes the last
  word, `⌃U` clears the whole query, no need to backspace character-by-character.
- Both changes ship together (one minor version bump), with no regressions to
  v0.1 behavior.

### Non-goals
- **Homebrew core** submission (`brew install keybook` directly). Requires
  meeting their notability bar; revisit after the project has traction.
- **Standalone binary** (`pkg` / `bun build --compile`). Removes the Node
  dependency at install time; defer until non-Node users start asking.
- **Mid-string editing** (cursor model, arrow-key navigation, forward delete).
  Captured as Phase B; ship Phase A first, then decide based on feel.
- Any change to the entry schema, the YAML loader, the search engine, the
  preview pane, or `keybook check`.

## 3. Feature 1 — Homebrew tap

### 3.1 Why a tap (not homebrew-core)

A **tap** is a personal Homebrew formula repository: a separate GitHub repo
named `homebrew-<name>` that anyone can add via `brew tap`. It needs no
approval, no notability bar, and the formula author owns updates.

End-user experience:

```bash
brew tap Ariyapong/tap
brew install keybook        # both `keybook` and `kb` are linked into PATH
```

Or in one line: `brew install ariyapong/tap/keybook`. `brew` resolves the
short form to the tap automatically.

### 3.2 New repo: `Ariyapong/homebrew-tap`

Sibling to the main `keybook` repo. Layout:

```
homebrew-tap/
├── README.md            # short — what the tap is and how to use it
└── Formula/
    └── keybook.rb
```

`README.md` is two paragraphs: the install command and a pointer back to
github.com/Ariyapong/keybook for the actual project.

### 3.3 The formula

```ruby
class Keybook < Formula
  desc "macOS TUI for searching keyboard shortcuts and recipes"
  homepage "https://github.com/Ariyapong/keybook"
  url "https://registry.npmjs.org/@arthony/keybook/-/keybook-0.1.2.tgz"
  sha256 "<sha256 of the tarball above>"
  license "MIT"

  depends_on "node@22"

  def install
    system "npm", "install", *Language::Node.std_npm_install_args(libexec)
    bin.install_symlink Dir["#{libexec}/bin/*"]
  end

  test do
    assert_match version.to_s, shell_output("#{bin}/keybook --version")
    system bin/"keybook", "check"
  end
end
```

Notes:
- **Source is the npm tarball**, not the GitHub release. This guarantees the
  installed code matches what `npm` users get, and the tarball is already
  pinned by version.
- **`node@22`** is pinned to match the project's `engines.node` and `.nvmrc`.
  Bumping the project's Node major bumps this line too.
- **`Language::Node.std_npm_install_args(libexec)`** installs into Homebrew's
  per-formula prefix, so the formula is self-contained.
- **`bin.install_symlink Dir["#{libexec}/bin/*"]`** links both `keybook` and
  `kb` from `package.json` `bin` into the user's PATH.
- **`test do`** runs in `brew test keybook` — sanity-checks the install by
  matching `--version` against the formula version, then runs `keybook check`
  as an end-to-end seed validation.

### 3.4 Release workflow (manual for v0.2)

When a new keybook version is published to npm:

1. Get the new tarball SHA256:
   ```bash
   curl -sL "https://registry.npmjs.org/@arthony/keybook/-/keybook-<NEW>.tgz" | shasum -a 256
   ```
2. In the tap repo, edit `Formula/keybook.rb`: bump the `url` version and the
   `sha256`.
3. Commit (`chore(formula): keybook <NEW>`) and push.
4. Verify locally: `brew untap Ariyapong/tap && brew install ariyapong/tap/keybook && brew test keybook`.

Automation (GitHub Action with `brew bump-formula-pr`) is a v0.3+ candidate.

### 3.5 README & advertise

- Update the main keybook README's **Install** section to list `brew install ariyapong/tap/keybook` **first** (it's the friction-free path for macOS users) and `npm install -g @arthony/keybook` second.
- Mention the tap once in the v0.2 GitHub Release notes.

## 4. Feature 2 — Readline-style search input (Phase A)

### 4.1 The problem

In v0.1, the search input only supports two edits: append a character and pop
the last character. To remove "delete" from `terminal delete`, the user has to
press Backspace seven times. No terminal text field works this way; the
inconsistency feels annoying.

Phase A closes this gap with the **minimum** set of bindings — no
cursor-position state, no rendering changes — so the diff is small and tightly
scoped. Mid-string editing (Phase B) is explicitly deferred.

### 4.2 Bindings added

| Key | Effect | Convention |
|---|---|---|
| `⌥⌫` (Option+Backspace) | Delete the previous word | macOS Cocoa default |
| `⌃W` | Delete the previous word | Unix / readline universal |
| `⌃U` | Clear the entire query | readline (cut to start of line) |

The two delete-word bindings are intentional duplicates: `⌥⌫` is what most
macOS users will reach for; `⌃W` works on every terminal even when Option is
sending native characters (no "Use Option as Meta key" required).

**Unchanged:** `⌫` / `Delete` still delete one character. `⌃C` and `⎋` still
quit. `↑` / `↓` / `⌃P` / `⌃N` still move selection. `⏎` still copies.

### 4.3 Implementation sketch

**Files touched:**
- `src/tui/App.tsx` — add three branches inside the existing `useInput` handler.
- `src/tui/input.ts` — **new** small pure module exporting `deleteWordBack(query: string): string`. Keeping it out of `App.tsx` makes it unit-testable.
- `src/tui/Footer.tsx` — add a single hint to the footer key-cap line.
- `tests/input.test.ts` — **new** unit tests for `deleteWordBack`.
- `tests/app.test.tsx` — add component-level cases for the three bindings.

**`deleteWordBack` semantics** (mirrors zsh/readline):
- Trailing whitespace is dropped first (a single press eats both the spaces and the word before them, matching readline).
- Then the run of non-whitespace immediately before the cursor is dropped.
- Empty query → returns empty query (no-op).

Examples (cursor `▏` at end):

| Input | After `⌥⌫` |
|---|---|
| `terminal delete▏` | `terminal ▏` |
| `terminal ▏` | `▏` (one press clears both word and trailing space) |
| `▏` | `▏` |
| `tmux pane split▏` | `tmux pane ▏` |

**Binding order in `useInput`:** the new branches must come **before** the
existing `if (key.backspace || key.delete)` line, so `⌥⌫` is routed to the
word-delete path rather than the single-char path.

```ts
// new — must precede the existing backspace branch
if ((key.meta && key.backspace) || (key.ctrl && input === "w")) {
  setQuery(deleteWordBack);
  setSelected(0);
  return;
}
if (key.ctrl && input === "u") {
  setQuery("");
  setSelected(0);
  return;
}
// existing — unchanged
if (key.backspace || key.delete) {
  setQuery((q) => q.slice(0, -1));
  ...
}
```

The existing fallthrough `if (input && !key.ctrl && !key.meta)` already
excludes `⌃W`, `⌃U`, `⌥<anything>` from being typed as a character. No change
needed there.

### 4.4 Discoverability — footer hint

`src/tui/Footer.tsx` currently shows `↑↓ move ⏎ copy ⎋ quit`. Add the new
bindings as a second column (or appended, depending on terminal width):

```
↑↓ move   ⏎ copy   ⎋ quit   ⌥⌫/⌃W del word   ⌃U clear
```

If the footer width would overflow, drop the `⌥⌫/⌃W` alias to just `⌃W` (the
universally-working one). The footer renders against `stdout.columns` already
via `layout.ts`; reuse the same width signal.

### 4.5 Tests

**`tests/input.test.ts`** (new, unit, pure):
- Returns empty string for empty input.
- Drops a single trailing word with no trailing whitespace.
- Drops trailing whitespace **and** the word before it (single-press semantics).
- Idempotent when the input is only whitespace (returns empty string).
- Multi-space sequences collapse correctly.

**`tests/app.test.tsx`** (new cases):
- Typing `terminal delete` then pressing `⌥⌫` leaves the query as `terminal `.
- Pressing `⌃W` produces the same result.
- Pressing `⌃U` after any query empties the query.
- After any of the three, `selected` is reset to 0 (matches existing
  same-on-typing behavior).
- A plain `⌫` still deletes one character (regression guard).

### 4.6 Compatibility note (no code change, but document)

In **Terminal.app**, the Option key sends native characters by default (`⌥A`
types `å`), which means `⌥⌫` may not register as Meta+Backspace unless the
user has enabled "Use Option as Meta key" in their profile. **iTerm2** sends
Option as Meta out of the box. The `⌃W` fallback covers users on either
terminal without configuration. README should mention this in a short
"Terminal.app users" note.

## 6. Acceptance criteria

A v0.2 is done when **all** of the following hold.

### Distribution
- [ ] `Ariyapong/homebrew-tap` exists on GitHub with `Formula/keybook.rb` and a short README.
- [ ] On a fresh Mac (or `brew untap Ariyapong/tap` then re-tap): `brew install ariyapong/tap/keybook` succeeds, installs `keybook` and `kb`, and `keybook --version` matches the published npm version.
- [ ] `brew test keybook` passes (asserts version + runs `keybook check` on the seed).
- [ ] The main keybook README's Install section leads with the Homebrew command.

### Input
- [ ] In the TUI, typing `terminal delete` then pressing `⌥⌫` leaves the query as `terminal ` (trailing space gone after the next char input; this is acceptable).
- [ ] `⌃W` produces the same result as `⌥⌫`.
- [ ] `⌃U` empties the query in one keystroke.
- [ ] Plain `⌫` still deletes a single character (regression).
- [ ] Footer hints at the new bindings.

### Quality
- [ ] All v0.1 tests still pass.
- [ ] New unit tests for `deleteWordBack` cover the cases in §4.5.
- [ ] New component tests in `app.test.tsx` cover the three bindings + regression.
- [ ] `pnpm typecheck`, `pnpm lint`, and `pnpm test` are all green in CI.
- [ ] No new runtime dependencies added to `package.json`.

### Release
- [ ] Version bumped to `0.2.0` and tagged `v0.2.0`.
- [ ] Published to npm as `@arthony/keybook@0.2.0`.
- [ ] GitHub Release notes summarize both features.
- [ ] Tap formula updated to point at `0.2.0` with the matching `sha256`.

## 7. Risks & open questions

| Item | Risk | Mitigation |
|---|---|---|
| Terminal.app users without "Option as Meta" don't get `⌥⌫` | Medium — most likely first complaint | `⌃W` works regardless; document in README |
| Brew formula tarball SHA needs manual update on each release | Low (manual) | Scripted in the release workflow; `brew bump-formula-pr` automation is a v0.3 candidate |
| `Language::Node.std_npm_install_args` API changes in future Homebrew versions | Low | The helper is stable; if it ever changes, the formula moves to an explicit `npm install --prefix` |
| Word-delete semantics surprise a vi-trained user (expects different cursor model) | Low | We're explicitly shipping the readline (emacs-style) convention — consistent with terminal default; vi-mode is out of scope |
| Footer hint string overflows on narrow terminals | Low | Width-aware via `layout.ts`; degrade to the shortest form |

## 8. Deferred to v0.3+

Captured here so they aren't lost. Decided against for v0.2 to keep the
release tight.

| Item | Notes |
|---|---|
| **Phase B readline** (cursor model) | `←/→`, `⌥←/⌥→`, `⌃A/⌃E`, `⌃D/⌥D`, `⌃K`. Requires cursor state in `App.tsx` and a re-render of `SearchInput.tsx`. Reassess after a week of Phase A use. |
| **Favorites** (`★` toggle) | Persist a small file in the data dir; boost favorites in search ranking. |
| **App filter** | A modifier key (e.g. `⌃F`) opens an app picker; subsequent searches are scoped to that app. |
| **`keybook add`** | Interactive Ink form to append a new entry to the right YAML file without leaving the TUI. |
| **More seed apps** | Chrome, Slack, Cursor / JetBrains, Sublime, Spotify, Notion. Content-only, follows the seed-expansion-spec template. |
| **Homebrew core submission** | Once stars / downloads cross their bar; switch end users from the tap to the bare formula. |
| **Standalone binary** | Node 22 SEA or `bun build --compile`. Adds ~30 MB but removes the Node dependency. |
| **Auto-bump tap on npm release** | GitHub Action using `brew bump-formula-pr` once the manual flow has been run a few times. |
